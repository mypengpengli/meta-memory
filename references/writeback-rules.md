# 记忆写回规则

默认不要把每一轮对话都直接写成长期记忆。

推荐顺序：

1. 先记录原始事件
2. 再写 `session`
3. 拿不准时写 `candidate`
4. 只有足够稳定时才提升到长期层

适合直接进入长期层的内容：

- 长期稳定身份和偏好
- 最近一段时间持续成立的状态
- 有明确时间点的关键事件
- 稳定关系模式
- 明确目标和项目约束
- 在某领域反复成立的经验

适合只进 `candidate` 的内容：

- 只出现过一次
- 还没验证
- 时间边界不清楚
- 可能和旧记忆冲突

不要直接写进长期层的内容：

- 一次性闲聊
- 未验证猜测
- 很长的原始日志
- 已经被新记忆覆盖的旧说法
