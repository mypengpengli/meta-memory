# 记忆系统说明

这套 Skill 里的“记忆”，不是单一的一份总文档。
它至少有三个维度：

- 领域：工作、学习、家庭与子女教育、日常生活、健康、财务与消费、人际与关系
- 性质：固定、领域、主题、项目、会话、候选、归档
- 主题网络：标签、关联主题、关联项目、关联来源

## 你可以先把它理解成 6 类性质

### 1. 固定记忆

存最稳定的长期偏好和长期规则。
例如：

- 回答风格偏好
- 长期做事习惯
- 长期不想踩的坑

### 2. 领域记忆

按大领域组织的长期背景。
例如：

- 工作领域的一般偏好
- 日常生活中的稳定习惯
- 家庭与子女教育中的总体理念

### 3. 主题记忆

按主题分类的长期经验。
例如：

- 编程相关经验
- 产品相关经验
- 业务/买卖相关经验

### 4. 项目记忆

只对当前项目生效的长期上下文。
例如：

- 当前项目的结构
- 当前项目里已经定下来的路线
- 当前项目里踩过的坑

### 5. 会话记忆

只服务当前这段工作的上下文。
例如：

- 这一轮做到哪了
- 当前临时结论
- 还没验证完的问题

### 6. 归档索引

只是“找到原始材料”的入口。
例如：

- 某次日志
- 某次原始聊天
- 某个很长的配置记录

归档索引默认不直接进 prompt。

## 正常加载时机

### 固定记忆

几乎每次都可以考虑，但内容必须很小。

### 领域记忆

当问题明显落在某个大领域时才加载。

### 主题记忆

当用户问题明显落到某个主题时才加载。

### 项目记忆

当前对话明显在讨论当前项目时才加载。

### 会话记忆

当前任务明显是在继续上一段工作时才加载。

### 归档索引

只有摘要不够、必须追证据时才使用。
