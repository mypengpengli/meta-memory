# 记忆系统说明

这套系统的目标，是长期维护“一个人”的记忆，而不是维护一堆分散主题。

因此它至少有四个维度：

- 主对象：某个人
- 时间：长期稳定 / 当前阶段 / 历史事件 / 已失效
- 网络：关系、目标、事件、来源
- 层级：画像、状态、事件、关系、目标、领域、会话、候选、归档

## 1. 人物画像

存最稳定、跨阶段仍成立的内容，例如：

- 身份与角色
- 稳定偏好
- 长期约束
- 核心习惯

## 2. 当前状态

存“最近一段时间”成立的阶段性状态，例如：

- 工作状态
- 健康状态
- 家庭状态
- 情绪状态

## 3. 事件记忆

存有明确时间边界的重要事件，例如：

- 转折点
- 关键经历
- 导致状态变化的节点

## 4. 关系记忆

存重要人物和关系模式，例如：

- 关系角色
- 沟通方式
- 边界
- 敏感点

## 5. 目标与项目

存持续推进的事项，例如：

- 长期目标
- 阶段计划
- 正在做的项目
- 已确定的项目约束

## 6. 领域切面

工作、学习、健康、财务这些仍然保留，但它们不再是主入口，只是人物的切面。

## 7. 会话记忆

只服务当前这段工作，例如：

- 当前做到哪
- 临时结论
- 下一步

## 8. 候选池

存可能重要、但还不能直接升级到长期层的内容，例如：

- 只出现过一次
- 还没验证
- 可能与旧记忆冲突

## 9. 归档

只是找到原始材料的入口，例如：

- 原始聊天
- 原始日志
- 长配置记录

归档默认不直接进 prompt。

## 每条记忆至少应表达什么

- 主对象是谁
- 这是画像、状态、事件还是关系
- 它从什么时候开始成立
- 到什么时候结束或失效
- 可信度如何
- 来源是什么
- 它和哪些人物、事件、目标有关
- 它是否被新版本覆盖
