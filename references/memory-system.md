# 记忆系统

这套系统围绕“一个具体的人”组织记忆，而不是围绕一堆分散主题。
这份文件解释层级，不负责回答前的读取顺序。

核心层级：

- `profile`
  - 最稳定的人物画像、长期偏好、长期约束
- `states`
  - 最近一段时间持续成立的阶段状态
- `events`
  - 有明确时间边界的关键经历
- `relationships`
  - 重要人物和稳定关系模式
- `goals`
  - 长期目标、计划和项目
- `domains`
  - 工作、学习、健康、财务等领域切面
- `sessions`
  - 当前回合或当前任务的临时上下文
- `candidates`
  - 可能重要但还不够稳定的内容
- `archive`
  - 原始聊天、日志、证据材料

判断一条记忆时，至少要能回答：

- 这是关于谁的
- 它属于哪一层
- 它从什么时候开始成立
- 它现在是否仍成立
- 它的可信度有多高
- 它来自哪里

存储方式：

- Markdown：正式记忆
- SQLite：原始事件、索引、处理状态、来源映射

默认原则：

- 长期记忆尽量短、稳、可复用
- 不确定内容先进入 `candidate`
- 原始归档只做证据层，不直接大量进入 prompt
