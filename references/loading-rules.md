# 记忆加载规则

回答前先判断：当前问题到底是在问“这个人”的哪一部分。

默认顺序：

1. `person-profile.md`
2. `state-current.md`
3. `goals-projects.md`
4. `relationships-current.md`
5. `timeline-index.md`
6. `domains-index.md`
7. `session-current.md`
8. `candidate-pool.md`
9. `archive-index.md`

加载原则：

- 先读当前有效信息，再读历史信息
- 先读人物核心，再读领域切面
- 先读摘要，再展开正文
- 不要把所有记忆一次性读进来

什么时候停止展开：

- 已经能回答当前问题
- 当前问题不涉及关系、事件或领域
- 原始证据对当前回答没有增益

什么时候才下钻原始层：

- 需要证据
- 需要时间线
- 当前正式记忆互相冲突
- 用户明确要求回看原始记录
